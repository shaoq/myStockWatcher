## Why

每日报告页面的日期选择功能存在三个影响用户体验的问题：

1. **用户可以选择未来日期** - DatePicker 允许选择今天之后的日期，但未来没有交易数据
2. **交易日判断可能失败** - 后端从 AkShare 获取交易日历时可能因网络或依赖问题失败，导致前端无法正确判断是否为交易日
3. **缺少报告生成引导** - 用户选择历史日期但该日期没有快照时，系统没有提供生成报告的选项，用户不知道如何获取历史报告

这些问题导致用户困惑和操作中断，需要修复以提升用户体验。

## What Changes

### 前端修改

1. **禁用未来日期选择**
   - 修改 `DatePicker` 的 `disabledDate` 函数，禁止选择今天之后的日期

2. **历史日期无报告时弹出确认对话框**
   - 当用户选择的交易日没有快照数据时，使用 `Modal.confirm` 询问是否生成报告
   - 确认后自动调用生成快照 API

3. **改进交易日状态显示**
   - 优化交易日历 API 调用的错误处理
   - 显示更清晰的状态信息

### 后端修改

1. **增强交易日历获取的健壮性**
   - 改进 `fetch_trading_calendar_from_akshare` 的错误处理和日志
   - 确保在 AkShare 不可用时提供友好的降级方案

## Capabilities

### New Capabilities

无新能力引入。

### Modified Capabilities

- `daily-report`: 增加日期选择限制和报告生成引导功能
  - 新增：日期选择器禁用未来日期
  - 新增：无快照历史日期的确认对话框交互
  - 修改：优化交易日判断的错误处理

- 交易日判断失败时有更清晰的错误提示
